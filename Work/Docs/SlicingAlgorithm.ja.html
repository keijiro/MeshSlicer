MESHSLICER / RUNTIME — アルゴリズムノート

平面によるメッシュ切断の仕組み

三角形の分類・クリップから、切り口を塞ぐキャップ生成まで

MeshSlicer は、水密(2-manifold)なメッシュを無限平面で切断し、切り口に蓋(キャップ)をした 2 つの閉じたメッシュを返すライブラリです。本稿では Slicer.Slice(Mesh, Plane) が 内部で行っている基本的な処理を、パイプラインの順に沿って解説します。

対象コード: Assets/MeshSlicer/Runtime/(Slicer.cs / Vertex.cs / CapBuilder.cs / PolygonTriangulator.cs ほか)。 Burst による高速化実装(BurstSlicer.cs)は本稿の範囲外です。

1入力と出力

入力は切断対象の Mesh と、メッシュのローカル空間で定義された平面 (法線 n と距離 d)です。出力は SliceResult で、 法線が指す側の破片 Positive と、その反対側の破片 Negative の 2 つのメッシュを持ちます。 メッシュ全体が片側に収まっていた場合、もう一方は null になります。

2処理の流れ

全体のパイプラインは次の 6 段階です。

1頂点の符号付き距離を計算
→
2切断位置のプローブとオフセット
→
3三角形を分類・分割
→
4切断線分を収集
→
5ループ化して三角形分割
→
6キャップを両側に追加

中核となるのは 3〜6 の「三角形を平面で切り分け、切り口を蓋する」処理です。 ステップ 2 の「オフセット」は、平面がちょうど頂点や辺の上を通ってしまう 縮退ケースを避けるための前処理で、第 6 節で詳しく説明します。

3符号付き距離による頂点の分類

まず全頂点について、平面からの符号付き距離 d = dot(n, p) + plane.d を計算します。符号が正なら法線側、 負なら反対側です。ただしそのまま使うのではなく、平面のごく近く (許容誤差 ε 以内)にある頂点は距離を 0 に丸めて 「平面上の頂点」として扱います。 

static float SignedDist(float3 p, float3 n, float d, float eps)
{
    var s = math.dot(n, p) + d;
    return math.abs(s) <= eps ? 0f : s;   // 平面近傍は「平面上」に丸める
}

ε はモデルの大きさに比例して決まります (ε = max(バウンディングボックス最大辺 × 10⁻⁴, 10⁻⁶))。 浮動小数点誤差で「わずかに正/わずかに負」と判定がぶれる頂点を、 一貫して平面上として扱うためのしきい値です。

三角形は 3 頂点の距離の符号によって、次の 3 通りに振り分けられます。

  • すべて d ≥ 0 — そのまま Positive 側へコピー
  • すべて d ≤ 0 — そのまま Negative 側へコピー
  • 正と負が混在 — 平面をまたいでいるので分割(次節)
  • 3 頂点すべてが d = 0(平面と完全に一致する面) — 厚みのない縮退面なので破棄

4またぐ三角形の分割

平面をまたぐ三角形は、Sutherland–Hodgman 法で 2 つの半空間それぞれに クリップします。三角形の 3 辺を順にたどり、「内側の頂点はそのまま出力、 平面を横切る辺は交点を挿入」という規則で切り抜くと、片側は三角形または四角形の 凸多角形になります。四角形になった場合は扇状(ファン)に 2 枚の三角形へ分けます。

なお d = 0(平面上)の頂点は両側の多角形に含めるのがポイントです。 これにより、頂点が平面上に乗っている三角形も隙間なく 2 つに分かれます。

交点での頂点属性の補間

辺と平面の交点位置は、両端の符号付き距離から比率 t = dₐ / (dₐ − d_b) で求まります。新しい頂点は位置だけでなく、 法線・接線・UV も同じ t で線形補間して生成します(Vertex.Lerp)。 補間した法線と接線は長さが 1 でなくなるため再正規化します。 これにより切断後もシェーディングやテクスチャが元のメッシュと連続します。

5キャップ(切断面)の生成

ここまでの処理で破片の「壁」はできますが、切り口は開いたままです。 分割時に集めておいた切断線分(図 2・図 3 の太い線分)から、 切り口の断面形状を復元して蓋を張ります。CapBuilder の仕事は次の 3 段階です。

① 端点の溶接と重複除去

各三角形から独立に生成された線分の端点は、浮動小数点誤差でごくわずかにずれています。 そこで座標をモデルサイズ比 10⁻⁵ 程度のグリッドに量子化し、同じセルに落ちた端点を 同一の点として溶接(weld)します。また、平面上に乗った辺は隣接する 2 つの三角形の両方から同じ線分として報告されるため、重複した線分は除去します。

② 閉ループへの連結

溶接後の線分は端点を共有するチェーンになっているので、端から順にたどって 閉ループに連結します。水密なメッシュを平面で切った断面は、 必ず 1 つ以上の閉じた輪郭(外周のみ、または外周+穴)になります。

③ 2D 投影と三角形分割

ループ上の点を平面上の 2 軸 (u, v) に投影して 2D 多角形にします (数値誤差対策として単位サイズに正規化もします)。その後の三角形分割は PolygonTriangulator が担当します。

  • 外周と穴の判定 — 各ループが他のいくつのループに含まれるかを数え、 偶数重なら外周(ソリッド)、奇数重なら穴と判定します(even-odd 規則)。 これによりドーナツ状や入れ子の断面も正しく扱えます。
  • 穴のブリッジ — 穴ループは、互いに見通せる頂点のペアを見つけて 外周へブリッジ辺でつなぎ、穴のない 1 本の多角形に変形します (図 5 ③ の紫の辺。行き帰り 2 重の辺として挿入されます)。
  • 耳切り法(ear clipping) — 凸な角から順に三角形を切り落として いく古典的な方法で多角形全体を三角形化します。

キャップを両側の破片へ追加

得られた三角形群は、同じ形状のまま両方の破片に追加されます。 ただし向きが逆になります。Positive 側の破片から見ると 切り口は法線と反対の方向を向くため、キャップの法線は −n、巻き順も反転。 Negative 側は法線 +n でそのまま、という対応です(図 1 参照)。 キャップの頂点には平面内の軸から作った接線と、UV = (0, 0) が割り当てられます。

6切断面のオフセット — 縮退の回避

ここまでの手順は、平面が三角形の辺を「きれいに横断」している限り正しく動きます。 ところが平面がちょうど頂点の上や辺の上を通ると話が変わります。 交線が 1 点に潰れたり、線分が途切れたり分岐したりして、切断境界が閉ループに ならなくなるのです。境界が閉じていないと三角形分割が破綻し、 キャップに穴が空くなどの破損が起きます。

そこで MeshSlicer は、本番の切断を行う前に切断位置をプローブ(下見)します。 切断線分の収集だけを軽量に実行し(クリップやメッシュ構築はしない)、 溶接後の境界グラフを検査するのです。判定基準はシンプルで、 すべての点の次数(接続する線分の数)がちょうど 2 であること。 これは境界が単純な閉ループの集まりであることと同値です。

プローブに失敗した場合は、平面を法線方向に m × ε だけずらして再試行します。 候補となる倍率 m は次の通りで、0 から始めて両側へ交互に、段階的に広げていきます。 

// 縮退時に試すオフセット量(ε 単位)。両側・段階的に拡大していく
static readonly float[] NudgeSteps = { 0, 1, -1, 3, -3, 8, -8, 24, -24, 72, -72, 216, -216 };

最大の 216ε でもモデルサイズの約 2% にすぎないため、切断位置の変化は見た目には ほぼ分かりません。ほとんどのケースは最初の 1〜2 ステップで解決します。 プローブは線分収集だけの軽い O(三角形数) パスなので、縮退のない通常の切断で 追加されるコストはわずかです。また、プローブと本番の切断は交線の計算コード (TryCutSegment)を共有しているため、「プローブで合格した境界」と 「実際に切断したときの境界」が食い違うことはありません。 すべての候補が失敗した場合は、元の位置でベストエフォートの切断にフォールバックします。

7まとめ — ソースコードとの対応

以上をまとめると、MeshSlicer の切断は「符号付き距離で三角形を仕分けし、 またぐものだけをクリップで分割、切り口の線分を閉ループに組み立てて蓋をする」 という流れです。数値誤差への備えとして、平面近傍の頂点の 0 スナップ(ε)、 端点の量子化溶接、そして切断位置そのもののオフセット探索という 3 段階の防御が入っています。

ファイル役割
Slicer.cs エントリポイント。距離計算、オフセット探索、三角形の分類とクリップ、全体の組み立て
Vertex.cs 位置・法線・接線・UV を持つ頂点と、その線形補間(Lerp)
CapBuilder.cs 切断線分の収集・溶接・ループ化、2D 投影、境界グラフの次数チェック
PolygonTriangulator.cs 穴あき多角形の三角形分割(外周/穴の判定、ブリッジ、耳切り法)
MeshBuilder.cs 三角形を蓄積して Mesh に変換
SliceResult.cs 結果(Positive / Negative の 2 メッシュ)

入力の前提は水密(2-manifold)なメッシュであることです。 開いたメッシュや自己交差のあるメッシュでは切断境界が閉ループにならないため、 キャップが正しく張れないことがあります(壁の分割自体は動作します)。
